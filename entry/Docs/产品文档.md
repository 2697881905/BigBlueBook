# 「大蓝书」产品文档

**版本**：V1.0  
**立项日期**：2026年7月9日  
**开发者**：个人开发者（个体工商户主体）  
**目标平台**：HarmonyOS NEXT（鸿蒙原生应用）  
**产品定位**：男性生活经验与处事方法论的分享社区


## 一、产品概述

### 1.1 产品使命

**「大蓝书」是一个专注于男性理性经验分享的社区平台。**

它的核心价值是：帮助男性用户在做消费决策、解决生活问题、提升个人能力时，能够找到真实、硬核、有方法论的参考信息。

### 1.2 与小红书的核心差异

| 维度 | 小红书 | 大蓝书 |
| :--- | :--- | :--- |
| **内容基调** | 美好生活、种草、情绪共鸣 | 硬干货、避坑、方法论、实操 |
| **内容形式** | 图文/视频混排，氛围感优先 | 结构化内容（优点/缺点/步骤） |
| **用户动机** | “逛一逛”、“被种草” | “搜一下”、“查攻略”、“做对比” |
| **互动方式** | 点赞、收藏、评论区夸夸 | 顶、抄作业、事实补充、理性辩论 |
| **社区氛围** | 精致、共情、温和 | 硬核、求真、有图有真相 |

### 1.3 目标用户画像

- **年龄**：18~35岁
- **性别**：男性（占比目标 > 85%）
- **特征**：
    - 有明确的消费决策需求（买数码、买车、买家电）
    - 喜欢自己动手解决问题（修东西、做饭、健身）
    - 关注个人成长（职场、理财、学习）
    - 反感“情绪化输出”，偏好“数据说话”
- **典型场景**：
    - “想买一款蓝牙耳机，先上大蓝书搜测评”
    - “家里水龙头漏水，上大蓝书看教程”
    - “刚入职一家新公司，上大蓝书学职场沟通”

## 五、业务流程

### 5.1 用户发布帖子的完整流程

```
用户进入发布器
    ↓
选择内容体裁（4选1）
    ↓
选择1~3个标签
    ↓
填写标题 + 正文 + 结构化字段 + 上传图片
    ↓
点击“发布”
    ↓
进入“待审核”队列（状态: pending）
    ↓
开发者人工审核（初期）
    ├─ 通过 → 对外可见（状态: published），计入首页流
    └─ 拒绝 → 通知用户，可修改后重发
```

### 5.2 内容分发流程

```
用户打开首页
    ↓
默认进入“热榜”Tab
    ├─ 按24小时“顶”数降序排列
    ├─ 展示前20条
    └─ 下拉加载更多
    ↓
切换到“最新”Tab
    ├─ 按发布时间倒序排列
    └─ 展示所有已审核帖子
    ↓
点击标签（如 #数码选购）
    ├─ 展示该标签下所有帖子
    └─ 默认按“最新”排序，可切换“最热”
```

### 5.3 互动激励闭环

```
用户阅读帖子后
    ├─ 觉得有用 → 点“顶”（帖子热度+1）
    ├─ 想以后再看 → 点“抄作业”（收藏+1）
    ├─ 有不同意见或补充 → 评论
    │   └─ 补充内容被其他用户“顶”得高 → 自动置顶
    └─ 觉得没用 → 划走（不产生互动）
```

> **设计理念**：不设“踩”或“反对”按钮。负面反馈通过“不互动”自然衰减，避免社区戾气。


## 六、UI设计规范（Apple极简风）

### 6.1 设计原则

1. **内容为王**：去除装饰性元素，所有UI服务于内容展示
2. **克制留白**：卡片间距≥16pt，保持呼吸感
3. **统一字体**：仅使用HarmonyOS Sans，层级清晰
4. **深色优先**：默认深色模式，契合男性用户偏好

### 6.2 色彩体系

## 七、合规与审核

### 7.1 主体资质

| 资质 | 要求 | 办理方式 |
| :--- | :--- | :--- |
| 个体工商户营业执照 | 上架鸿蒙应用必须 | 当地工商局/线上办理 |

| 安全评估报告 | 社交类应用必备 | 当地网信办提交 |

> **关键提醒**：个人开发者账号**无法上架UGC社交应用**，**必须用个体工商户或企业主体**注册鸿蒙开发者账号。

### 7.2 内容审核机制

**三道防线：**

| 防线 | 方式 | 说明 |
| :--- | :--- | :--- |
| **第一道：前置过滤** | 敏感词拦截 | 发布时实时检测，命中则提示修改 |
| **第二道：人工审核** | 所有帖子先审后发 | 初期日发帖<50篇，开发者自己审核 |
| **第三道：用户举报** | 举报优先处理 | 被举报内容48小时内处理 |

**敏感词库**：使用开源词库（如Google的Bad Word List），重点拦截：
- 涉政敏感词
- 色情低俗词
- 人身攻击词
- 男女对立引战词

### 7.3 社区公约（核心条款）

1. **禁止性别对立**：不得发布煽动男女对立的言论，违者删帖+封号
2. **禁止云评测**：消费类帖子建议附购买凭证或实拍图
3. **禁止纯情绪宣泄**：帖子必须有实质性内容（测评/教程/经验）
4. **禁止广告推广**：软文、带货链接一律删除
5. **尊重事实**：评论区禁止恶意杠精，鼓励“事实补充”

## 十、风险与应对

## 十一、总结

**大蓝书的核心竞争力不在于“反小红书”，而在于“为男性用户提供真实、硬核、有方法论的参考信息”。**

它是一个**理性驱动的社区**，不鼓励情绪宣泄，不制造性别对立，只关注：
- 买什么不踩坑
- 怎么做更高效
- 怎么混得更好

这30个标签、4种内容体裁、3个互动按钮，构成了“大蓝书”的完整产品骨架。
