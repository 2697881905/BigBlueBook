# 大蓝书 · 接口契约（冻结版 V1）

> 用途：前后端联调的唯一依据。字段、错误码、分页均以此为准。
> 后端实现：Express + TypeScript + Prisma（见 `backend/`）。
> 统一前缀：`/v1`

## 1. 通用规范

### 1.1 响应包
所有接口统一返回：
```json
{ "code": 0, "data": <T>, "message": "success" }
```
- `code === 0` 表示成功；非 0 表示业务失败，`message` 为错误描述。
- 鉴权失败除 `code` 外同时使用 HTTP 401 状态码。

### 1.2 鉴权
需要登录的接口在 Header 带：
```
Authorization: Bearer <token>
```
`token` 由 `POST /v1/auth/login` 下发。

### 1.3 错误码表
| code | HTTP | 含义 |
| :--- | :--- | :--- |
| 0 | 200 | 成功 |
| 400 | 400 | 参数错误（如缺少必填项） |
| 401 | 401 | 未登录 / 登录过期 |
| 403 | 403 | 无权限（如删他人帖子） |
| 404 | 404 | 资源不存在 |
| 409 | 409 | 冲突 |
| 500 | 500 | 服务内部错误 |

### 1.4 分页参数
列表类接口统一：
- `page`：页码，默认 1
- `limit`：每页数量，默认 20，最大 50
- 响应 `data.pagination`: `{ page, limit, total }`

### 1.5 数据约定
- 标签在 `posts.tags` 中存为**纯名称数组**（如 `["数码选购","手机数码"]`），前端展示时自行拼接 `#`。
- 图片以 **CDN URL 字符串** 存储；`images` 为 URL 数组，最多 9 张。
- `genre` 取值：`review`(测评报告) / `pitfall`(避坑指南) / `tutorial`(实操教程) / `debate`(辩论投票)。

## 2. 接口清单

### 2.1 登录 / 用户
| 方法 | 路径 | 鉴权 | 说明 |
| :--- | :--- | :--- | :--- |
| POST | `/v1/auth/login` | 否 | 鸿蒙账号授权登录（MVP 用 openId） |
| GET | `/v1/auth/me` | 是 | 当前用户信息 |
| PUT | `/v1/auth/me` | 是 | 更新昵称/头像 |

**POST /v1/auth/login**
```json
// 请求
{ "openId": "harmony-xxxx", "nickname": "科技老张", "avatar": "https://cdn.x/avatar.jpg" }
// 响应
{ "code": 0, "data": { "token": "<jwt>", "user": { "id": 1, "nickname": "科技老张", "avatar": "...", "gender": 1 } }, "message": "success" }
```

### 2.2 帖子
| 方法 | 路径 | 鉴权 | 说明 |
| :--- | :--- | :--- | :--- |
| GET | `/v1/posts` | 否 | 列表（分页+标签+排序） |
| GET | `/v1/posts/:id` | 否 | 详情（含评论） |
| POST | `/v1/posts` | 是 | 发布（进入待审核） |
| DELETE | `/v1/posts/:id` | 是 | 删除（仅本人） |

**GET /v1/posts**
```
/v1/posts?page=1&limit=20&sort=hot|latest|recommend&tag=数码选购
```
- `sort=hot`：按 24h 内 `upCount` 降序；`latest`：按时间倒序；`recommend`：P1 初期回退到最新。

```json
// 响应 data
{
  "list": [
    {
      "id": 1, "title": "iPhone 17 Pro 深度使用30天报告", "coverImage": "https://cdn.x/cover.jpg",
      "genre": "review", "tags": ["数码选购","手机数码"],
      "upCount": 236, "commentCount": 89, "bookmarkCount": 12,
      "createdAt": "2026-07-08T10:00:00Z",
      "user": { "id": 1, "nickname": "科技老张", "avatar": "https://cdn.x/avatar.jpg" }
    }
  ],
  "pagination": { "page": 1, "limit": 20, "total": 156 }
}
```

**POST /v1/posts**
```json
// 请求
{
  "title": "iPhone 17 Pro 深度使用30天报告",
  "content": "正文...",
  "coverImage": "https://cdn.x/cover.jpg",
  "images": ["https://cdn.x/1.jpg","https://cdn.x/2.jpg"],
  "genre": "review",
  "tags": ["数码选购","手机数码"],
  "structuredData": { "pros": "续航强", "cons": "偏重", "rating": 4.5, "targetAudience": "重度用户" }
}
// 说明：成功创建后 status=0（待审核），不会立即出现在 /v1/posts 列表
```

### 2.3 评论
| 方法 | 路径 | 鉴权 | 说明 |
| :--- | :--- | :--- | :--- |
| GET | `/v1/posts/:id/comments` | 否 | 评论列表 |
| POST | `/v1/posts/:id/comments` | 是 | 发布评论 |
| DELETE | `/v1/comments/:id` | 是 | 删除评论（仅本人） |

```json
// POST 请求
{ "content": "补充一个缺点...", "parentId": null, "isFact": 1 }
// isFact=1 表示"事实补充"，前端可高亮
```

### 2.4 互动
| 方法 | 路径 | 鉴权 | 说明 |
| :--- | :--- | :--- | :--- |
| POST | `/v1/posts/:id/up` | 是 | 顶帖子 |
| DELETE | `/v1/posts/:id/up` | 是 | 取消顶 |
| POST | `/v1/posts/:id/bookmark` | 是 | 收藏（抄作业） |
| DELETE | `/v1/posts/:id/bookmark` | 是 | 取消收藏 |

> 幂等：重复顶/收藏不会改变计数（唯一约束 userId+postId）。

### 2.5 标签 / 上传
| 方法 | 路径 | 鉴权 | 说明 |
| :--- | :--- | :--- | :--- |
| GET | `/v1/tags` | 否 | 全部标签（按 useCount 降序） |
| POST | `/v1/upload/token` | 是 | 获取 OSS/OBS 直传凭证 |

**GET /v1/tags** → `data: [{ "id":1, "name":"数码选购", "emoji":"📱", "category":"消费决策", "useCount":0 }, ...]`
