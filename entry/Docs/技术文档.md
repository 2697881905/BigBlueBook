# 「大蓝书」技术文档

**版本**：V1.0  
**目标**：完成MVP版本的产品开发与内部测试  
**暂不涉及**：上线部署、商业化、大规模运维


## 一、整体技术架构

采用**前后端分离**架构：

```
┌─────────────────────────────────────────────────────────┐
│                    前端（鸿蒙原生）                       │
│  语言：ArkTS  |  UI框架：ArkUI  |  IDE：DevEco Studio   │
└─────────────────────────────────────────────────────────┘
                              ↓ HTTPS
┌─────────────────────────────────────────────────────────┐
│                   后端 API 服务                          │
│  语言：TypeScript / Node.js  |  框架：Express / NestJS   │
└─────────────────────────────────────────────────────────┘
                              ↓
┌──────────────────┬──────────────────┬──────────────────┐
│    MySQL         │    Redis         │   OSS 对象存储    │
│  (关系数据)       │   (缓存/会话)     │   (图片/文件)     │
└──────────────────┴──────────────────┴──────────────────┘
```


## 二、前端技术栈（鸿蒙原生）

### 2.1 核心选型

| 项目 | 选型 | 说明 |
| :--- | :--- | :--- |
| **开发平台** | HarmonyOS NEXT（纯血鸿蒙） | 原生应用，不兼容安卓 |
| **开发语言** | ArkTS | 鸿蒙官方推荐语言，基于TypeScript扩展 |
| **UI框架** | ArkUI | 声明式UI框架，类似SwiftUI/Compose |
| **应用模型** | Stage模型 | 鸿蒙官方推荐的应用开发模型 |
| **IDE** | DevEco Studio | 最新稳定版 |

### 2.2 项目模板选择

创建项目时选择 **ArkUI-X** 模板：
- **理由**：虽然当前只上架鸿蒙，但ArkUI-X保留未来跨平台（Android/iOS）的可能性，避免技术债务
- **不选**：Native C++模板（大蓝书不需要高性能C/C++计算）

### 2.3 前端核心依赖与模块

| 模块 | 用途 | 鸿蒙API |
| :--- | :--- | :--- |
| **网络请求** | 调用后端API | `@ohos.net.http` 或 RCP（Remote Communication Platform） |
| **图片选择** | 从相册选取图片 | `PhotoViewPicker` |
| **图片上传** | 上传到OSS | `@ohos.request` 模块 |
| **本地存储** | 用户偏好/缓存 | `Preferences`（轻量级键值对） |
| **本地数据库** | 离线数据缓存 | `ArkData` 关系型数据库（基于SQLite） |
| **路由导航** | 页面跳转 | `router` 模块 |
| **状态管理** | 跨组件状态 | `@State` / `@AppStorage` / `PersistentStorage` |

## 三、后端技术栈

### 3.1 核心选型

| 项目 | 选型 | 理由 |
| :--- | :--- | :--- |
| **运行环境** | Node.js 20+ | 与前端ArkTS同属TypeScript生态，语言统一 |
| **Web框架** | Express 或 NestJS | Express轻量灵活；NestJS结构化适合长期维护 |
| **语言** | TypeScript | 类型安全，减少运行时错误 |
| **ORM** | Prisma | 类型安全的数据库ORM，支持MySQL |
| **API风格** | RESTful | 简单直接，适合MVP |

> **个人开发者建议**：优先选 **Express + TypeScript**，学习曲线平缓，开发速度快。等产品验证后再考虑迁移到NestJS。

## 四、数据库设计

### 4.1 选型

| 项目 | 选型 | 说明 |
| :--- | :--- | :--- |
| **主数据库** | MySQL 8.0 | 成熟稳定，个人开发者免费使用 |
| **缓存** | Redis（可选） | 用于会话存储、热点数据缓存，MVP阶段可暂不引入 |

### 4.2 核心表结构（Prisma Schema）

```prisma
// prisma/schema.prisma

generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "mysql"
  url      = env("DATABASE_URL")
}

// 用户表
model User {
  id          Int       @id @default(autoincrement())
  openId      String    @unique @db.VarChar(64)   // 鸿蒙账号ID
  nickname    String    @db.VarChar(32)
  avatar      String?   @db.VarChar(255)
  gender      Int?      @default(1)               // 1-男 2-女
  followTags  Json?                               // 关注的标签列表
  status      Int       @default(1)               // 1-正常 0-封禁
  createdAt   DateTime  @default(now())
  updatedAt   DateTime  @updatedAt
  
  posts       Post[]
  comments    Comment[]
  ups         Up[]
  bookmarks   Bookmark[]
}

// 帖子表
model Post {
  id              Int       @id @default(autoincrement())
  userId          Int
  user            User      @relation(fields: [userId], references: [id])
  title           String    @db.VarChar(100)
  content         String?   @db.Text
  coverImage      String?   @db.VarChar(255)
  images          Json?                              // 图片URL列表，最多9张
  genre           String    @db.VarChar(20)          // review | pitfall | tutorial | debate
  tags            Json                               // 标签列表，1~3个
  structuredData  Json?                              // 结构化字段（体裁不同，内容不同）
  upCount         Int       @default(0)
  bookmarkCount   Int       @default(0)
  commentCount    Int       @default(0)
  status          Int       @default(0)              // 0-待审核 1-已发布 2-已拒绝
  createdAt       DateTime  @default(now())
  updatedAt       DateTime  @updatedAt
  
  comments        Comment[]
  ups             Up[]
  bookmarks       Bookmark[]
}

// 评论表
model Comment {
  id          Int       @id @default(autoincrement())
  postId      Int
  post        Post      @relation(fields: [postId], references: [id])
  userId      Int
  user        User      @relation(fields: [userId], references: [id])
  parentId    Int?                                 // 0或null表示一级评论
  content     String    @db.Text
  isFact      Int       @default(0)                // 1-事实补充
  upCount     Int       @default(0)
  createdAt   DateTime  @default(now())
  updatedAt   DateTime  @updatedAt
}

// 顶（点赞）记录表
model Up {
  id          Int       @id @default(autoincrement())
  userId      Int
  user        User      @relation(fields: [userId], references: [id])
  postId      Int
  post        Post      @relation(fields: [postId], references: [id])
  createdAt   DateTime  @default(now())
  
  @@unique([userId, postId])                       // 同一用户对同一帖子只能顶一次
}

// 收藏（抄作业）记录表
model Bookmark {
  id          Int       @id @default(autoincrement())
  userId      Int
  user        User      @relation(fields: [userId], references: [id])
  postId      Int
  post        Post      @relation(fields: [postId], references: [id])
  createdAt   DateTime  @default(now())
  
  @@unique([userId, postId])
}

// 标签表
model Tag {
  id          Int       @id @default(autoincrement())
  name        String    @unique @db.VarChar(20)    // 如"数码选购"
  emoji       String    @db.VarChar(10)            // 如"📱"
  category    String    @db.VarChar(20)            // 消费决策/动手实操/个人成长/生活方式
  useCount    Int       @default(0)                // 被使用次数
  createdAt   DateTime  @default(now())
}
```


## 五、图片存储方案

### 5.1 选型

| 项目 | 选型 | 说明 |
| :--- | :--- | :--- |
| **对象存储** | 华为云OBS 或 阿里云OSS | 均有鸿蒙SDK适配 |
| **CDN加速** | 存储服务自带CDN | 图片加载更快 |

### 5.2 存储目录结构

```
/bigbluebook/
├── images/
│   ├── avatars/          # 用户头像
│   │   └── {userId}/...
│   ├── posts/            # 帖子图片
│   │   └── {postId}/
│   │       ├── cover.jpg
│   │       ├── img_1.jpg
│   │       └── img_2.jpg
│   └── temp/             # 临时文件（审核用）
└── ...
```

### 5.3 图片处理规则

| 规则 | 说明 |
| :--- | :--- |
| **上传前压缩** | 前端压缩，长边不超过2048px，质量80% |
| **格式** | 统一转为JPEG或WebP |
| **单张限制** | 不超过5MB |
| **数量限制** | 每帖最多9张图 |
| **访问方式** | 通过CDN URL直接访问，不经过业务服务器 |


## 六、API接口设计

### 6.1 接口规范

| 规范 | 说明 |
| :--- | :--- |
| **Base URL** | `https://api.bigbluebook.com/v1` |
| **请求格式** | JSON |
| **响应格式** | `{ code: 0, data: T, message: string }` |
| **鉴权方式** | Bearer Token（鸿蒙账号授权后下发） |

### 6.2 核心接口清单

| 接口 | 方法 | 路径 | 说明 |
| :--- | :--- | :--- | :--- |
| **用户** | | | |
| 登录/注册 | POST | `/auth/login` | 鸿蒙账号授权 |
| 获取个人信息 | GET | `/users/me` | 当前用户信息 |
| 更新个人信息 | PUT | `/users/me` | 昵称/头像等 |
| **帖子** | | | |
| 获取帖子列表 | GET | `/posts` | 支持分页、标签筛选、排序 |
| 获取帖子详情 | GET | `/posts/:id` | 含结构化数据 |
| 发布帖子 | POST | `/posts` | 进入待审核 |
| 删除帖子 | DELETE | `/posts/:id` | 仅本人可删 |
| **互动** | | | |
| 顶帖子 | POST | `/posts/:id/up` |  |
| 取消顶 | DELETE | `/posts/:id/up` |  |
| 收藏帖子 | POST | `/posts/:id/bookmark` |  |
| 取消收藏 | DELETE | `/posts/:id/bookmark` |  |
| **评论** | | | |
| 获取评论列表 | GET | `/posts/:id/comments` | 支持分页 |
| 发布评论 | POST | `/posts/:id/comments` |  |
| 删除评论 | DELETE | `/comments/:id` | 仅本人可删 |
| **标签** | | | |
| 获取所有标签 | GET | `/tags` | 按使用次数排序 |
| **上传** | | | |
| 获取上传凭证 | POST | `/upload/token` | 获取OSS临时令牌 |
| 图片上传回调 | POST | `/upload/callback` | 上传完成后通知后端 |

## 九、开发优先级总结

| 优先级 | 说明 |
| :--- | :--- |
| **P0** | 必须做，没有就无法验证产品核心价值 |
| **P1** | 重要但可后续迭代，MVP可先简化 |

**P0功能清单**：首页、帖子详情、发布器（4步流程）、图片上传、用户认证、帖子列表（热榜/最新）、标签系统、评论、顶/收藏、深色模式

**P1功能清单**：搜索、消息通知、个人主页完整版、推荐算法（初期可用简单规则替代）
