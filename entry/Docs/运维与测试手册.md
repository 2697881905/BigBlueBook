# 大蓝书 运维与测试手册

> 面向 Java/Android 背景的开发者，覆盖鸿蒙生命周期理解、环境搭建、后端运维、构建运行、问题排查与测试。

## 一、项目概览

| 层 | 技术栈 | 位置 |
|---|---|---|
| 鸿蒙前端 | HarmonyOS NEXT / ArkTS / ArkUI（Stage 模型，API 24） | `entry/` |
| 后端 | Node.js + Express + TypeScript + Prisma + MySQL | `backend/` |
| 包名 | `com.bigbluebook.app` | `AppScope/app.json5` |
| 入口 Ability | `EntryAbility` → 加载 `pages/Index` | `entry/src/main/ets/entryability/` |

目录结构：
- `entry/src/main/ets/pages/` — 5 个页面：Index(首页壳)/DetailPage/PublishPage/ProfilePage/MessagePage
- `entry/src/main/ets/components/` — 组件：HomeTab/CircleTab/PostCard/TagNav/CommentList/ImagePicker
- `entry/src/main/ets/services/api.ets` — HTTP 客户端 + 领域 API
- `entry/src/main/ets/utils/` — auth(登录态)/image(压缩)/nav(跨页传参)/format
- `entry/src/main/ets/models/types.ets` — 数据模型
- `backend/src/` — routes/services/middleware/config/prisma

## 二、鸿蒙应用生命周期（Java/Android 视角）

### 2.1 UIAbility 生命周期（类比 Activity）

HarmonyOS Stage 模型的 UIAbility 对应 Android 的 Activity，6 个回调按序触发：

| HarmonyOS 回调 | 触发时机 | Android 类比 | 本项目做了什么 |
|---|---|---|---|
| `onCreate` | Ability 实例创建 | `onCreate` | 设置深色模式 |
| `onWindowStageCreate` | 窗口创建，加载页面 | `setContentView` | `loadContent('pages/Index')` 加载首页 |
| `onForeground` | 进入前台，可见可交互 | `onResume` | （可刷新数据） |
| `onBackground` | 退到后台 | `onPause` | （可暂停耗时操作） |
| `onWindowStageDestroy` | 窗口销毁，释放 UI 资源 | `onDestroy` 前 | |
| `onDestroy` | 实例销毁 | `onDestroy` | |

流程：`onCreate → onWindowStageCreate → onForeground ↔ onBackground → onWindowStageDestroy → onDestroy`，其中 `onForeground`/`onBackground` 可来回切换。

### 2.2 页面生命周期（@Entry 组件）

每个 `@Entry` 装饰的页面有回调：

| 回调 | 时机 | 类比 |
|---|---|---|
| `aboutToAppear` | 页面创建后、build 前 | `onCreate` |
| `build` | 首次绘制 UI | `onCreateView` |
| `onPageShow` | 页面显示（切到前台/从其他页返回） | `onResume` |
| `onPageHide` | 页面隐藏 | `onPause` |
| `aboutToDisappear` | 页面销毁前 | `onDestroy` |

本项目 `HomeTab`/`ProfilePage` 在 `aboutToAppear` 里触发数据加载。

### 2.3 组件生命周期（@Component）

普通 `@Component` 只有 `aboutToAppear` / `aboutToDisappear`，**没有** `onPageShow`/`onPageHide`（那是 `@Entry` 专属）。

### 2.4 关键区别（Java 开发者易混点）

- **Ability ≠ Activity**：一个 UIAbility 可承载多页面（本项目单 Ability + Index 壳页内嵌多 Tab）。
- **没有 Manifest**：配置在 `module.json5`（不是 `AndroidManifest.xml`）。
- **路由**：`router.pushUrl` 已弃用（仅警告），推荐 `Navigation`/`NavPathStack`，本项目暂用 pushUrl。
- **跨页传参**：`router.getParams` 在 API 24 失效，本项目改用 `AppStorage`（全局单例）传 `detailPostId`。
- **状态管理**：`@State`/`@Prop`/`@Link`/`@Watch` 对应 Jetpack Compose 的 `remember`/`mutableStateOf`，但 ArkTS 严格模式限制更多（见排查章节）。

## 四、后端启动与验证

### 4.1 配置
复制 `backend/.env.example` 为 `backend/.env`，填入：
- `DATABASE_URL` — MySQL 连接串
- `JWT_SECRET` — JWT 密钥
- `COS_*` — 腾讯云 COS（图片上传，不填则上传接口返回 400，不影响起服）

### 4.2 建表（重要：用 db push，不要 migrate dev）

`bigbluebook` MySQL 用户无 `CREATE DATABASE` 权限，`prisma migrate dev` 会报 P3014/P1010（建不了 shadow 库）。统一用：

```
cd ~/HarmonyProject1/backend
npx prisma db push
```

### 4.3 灌种子数据

```
cd ~/HarmonyProject1/backend
npm run seed:tags
```

### 4.4 启动

```
cd ~/HarmonyProject1/backend
npm run dev
```

起成功打印 `BigBlueBook API listening on http://localhost:3000`。此终端保持常驻。

### 4.5 验证

```
curl -s --max-time 3 http://localhost:3000/health && echo " 后端OK" || echo " 后端没起"
```

返回 `{"ok":true}` 即正常。

## 五、前端构建与运行

### 5.1 DevEco Studio（推荐）
- 打开项目 → 选 `entry` 模块 → 点 Run（▶）
- 自动编译 + 安装 HAP 到模拟器/真机

### 5.2 命令行构建 HAP

```
cd ~/HarmonyProject1
DEVECO_SDK_HOME=/Applications/DevEco-Studio.app/Contents/sdk /Applications/DevEco-Studio.app/Contents/tools/node/bin/node /Applications/DevEco-Studio.app/Contents/tools/hvigor/bin/hvigorw.js --mode module -p module=entry@default -p product=default -p requiredDeviceType=phone assembleHap --analyze=normal --parallel
```

改过 `main_pages.json`（增删页面）后，hvigor daemon 会缓存旧页面表，需：

```
pkill -f hvigor
rm -rf .preview
```

再重新构建。

### 5.3 BASE_URL 配置（关键）

`entry/src/main/ets/services/api.ets` 第 8 行：

```
const BASE_URL = 'http://192.168.31.104:3000';
```

- **模拟器**：用 Mac 局域网 IP（`ipconfig getifaddr en0` 查）。`10.0.2.2` 是 Android 约定，HarmonyOS 模拟器不认。
- **真机**：同局域网 IP，手机与电脑同 WiFi。
- IP 变了（DHCP 重新分配）就改这里重编。
- 改完必须重新 Run/assembleHap，旧 HAP 里烤着旧地址。


## 七、常见问题排查

### 7.1 App 联网错误（NETSTACK 日志）

| errno | 含义 | 原因 | 处理 |
|---|---|---|---|
| 111 | ECONNREFUSED | 目标无服务监听 | 后端没起，`npm run dev` |
| 115 | EINPROGRESS | 连接进行中（非错误） | 瞬时可忽略；持续则是 IP 不对或防火墙挡 |
| timeout | 超时 | 防火墙挡 / IP 变了 / 后端没起 | 见下 |

排查顺序：
1. 宿主 `curl -s --max-time 3 http://localhost:3000/health` → 不通则起后端。
2. 宿主 `curl -s --max-time 3 http://<LAN_IP>:3000/health` → 不通：
   - IP 变了 → `ipconfig getifaddr en0` 查新 IP，改 BASE_URL 重编。
   - 防火墙 → 系统设置→网络→防火墙，关掉测；确认后在防火墙选项放行 node 入站 3000。
3. App 端仍超时 → DevEco 重新 Run 装 HAP（旧 HAP 烤旧地址）。

### 7.2 npm install 卡住/报错
- **卡住转圈**：Prisma 引擎二进制从官方 CDN 拉取慢。设 `PRISMA_ENGINES_MIRROR=https://registry.npmmirror.com/-/binary/prisma/` 后重装。
- **UNABLE_TO_GET_ISSUER_CERT_LOCALLY**：代理 HTTPS 拦截。`npm config set strict-ssl false`。
- **Cannot find module 'cos-nodejs-sdk-v5'**：漏 `npm install`，或 package.json 版本写错（正确 `^3.0.0`）。

### 7.3 Prisma migrate 报 P3014/P1010
`bigbluebook` 用户无 `CREATE DATABASE` 权限。**不要用 migrate dev**，改用 `prisma db push`。

### 7.4 ArkTS 编译错误

| 错误码 | 原因 | 修法 |
|---|---|---|
| 10505001 | `@State` 变量名与 ArkUI 保留成员冲突（如 `tabIndex`/`width`/`height`） | 改名（`tabIndex`→`tabIdx`） |
| 10605038 | 对象字面量未对应已声明 interface | 抽具名 interface 再用 |
| 10605074 | 禁止解构 `const {a,b}=obj` | 逐字段赋值 |
| 10505001 | `sendData` 不存在 | 改 `extraData` |

ArkTS 源码必须 `.ets` 扩展名，`.ts` import `.ets` 会报 "Importing ArkTS files in JS and TS files is forbidden"。

### 7.5 弃用警告（不阻断编译）
`showToast`/`pushUrl`/`back` 已弃用，仅 WARN。可后续统一迁移到 `promptAction.openToast`/`Navigation`。
